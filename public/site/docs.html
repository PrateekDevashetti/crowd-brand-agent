Quickstart

Authenticate with x-api-key. Everything async returns 202 with an id; poll with ?wait=true for a 55-second long-poll.

# 1 · Onboard a brand
curl -X POST localhost:3000/api/v1/brands \
  -H "x-api-key: $KEY" -H "content-type: application/json" \
  -d '{"url": "https://yourbrand.com"}'

# 2 · Poll until ready (profile + scraped logo)
curl localhost:3000/api/v1/brands/$BRAND_ID -H "x-api-key: $KEY"

# 3 · Generate on-brand
curl -X POST localhost:3000/api/v1/images/generations \
  -H "x-api-key: $KEY" -H "content-type: application/json" \
  -d '{"brandSessionId": "$BRAND_ID", "prompt": "Hero image for a spring launch", "aspectRatio": "16:9"}'

# 4 · Long-poll the result
curl "localhost:3000/api/v1/images/$IMAGE_ID?wait=true" -H "x-api-key: $KEY"

Endpoints

Method · PathWhat it does
POST /api/v1/brandsOnboard a brand from a URL (scrapes logo + imagery too)
GET /api/v1/brands · /brands/{id}List / fetch brands with profile and logoUrl
PATCH /api/v1/brands/{id}Update name or merge profile changes
POST /api/v1/images/generationsGenerate (variants, reference images, 1K/2K/4K)
POST /api/v1/images/{id}/edit · /resizeNatural-language edit · generative recompose
POST /api/v1/images/{id}/background-removal · /vectorizeScaffolds (swap in rembg / vtracer)
POST /api/v1/images/uploadsImport an image by URL
GET /api/v1/images?q=…List + plain-English search over prompts
GET /api/v1/images/{id}?wait=trueLong-poll an image until terminal state
GET /api/v1/account · /account/creditsAccount info · balance + ledger

Errors use a stable envelope: { defined, code, status, message, data } — e.g. BRAND_NOT_FOUND (422), INSUFFICIENT_CREDITS (402, includes action_url).

MCP

Streamable HTTP endpoint at POST /api/mcp, authenticated with Authorization: Bearer <key>. Ten tools:

  • bloom_onboard_brand, bloom_get_brand, bloom_list_brands, bloom_update_brand
  • bloom_generate_image, bloom_edit_image, bloom_resize_image, bloom_get_image
  • bloom_search_images, bloom_upload_image
claude mcp add --transport http crowd http://localhost:3000/api/mcp \
  --header "Authorization: Bearer $KEY"

Agent skill

A packaged skill teaches agents prompting rules, channel aspect ratios, and the full workflow. Download the .skill file and add it in Claude's Skills settings, or read the source at /skills/brandlayer/SKILL.md.

Templates

Copy-ready integration examples ship in the repo's examples/ folder:

  • newsletter-image — a fresh on-brand header for every email send
  • social-repurpose — one topic into consistent 16:9 / 1:1 / 9:16 images

Self-hosting

Node 20+, Postgres, Redis. docker compose up -d, npm install, npm run dev + npm run worker. The schema applies itself on boot. Set GEMINI_API_KEY or OPENAI_API_KEY for real generation (mock provider otherwise), and Clerk keys for auth. See DEPLOY.md for Railway.
