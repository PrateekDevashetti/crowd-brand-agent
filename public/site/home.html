FAQ

Frequently asked questions

What is Crowd?
Crowd is a self-hostable brand layer: it learns a brand from its website (palette, type, tone, logo, imagery rules), then injects that profile into every image you generate — so a one-line prompt comes back on-brand. It exposes the same capability as a REST API and an MCP server for agents.
How does onboarding work?
Paste a URL. Crowd scrapes the site, an LLM distills the signals into a structured brand profile, and the site's logo and imagery are imported into your library. It takes about 10–30 seconds and is a one-time setup per brand.
How do credits work?
A 2K image costs 1 credit and a 4K image costs 2, per variant. Edits and resizes are priced like generations at the source image's resolution. Failed jobs refund automatically.
Can I have multiple brands?
Yes — onboard as many brands as you like and switch between them from the dashboard. Agents address a specific brand with its session id.
What can agents do over MCP?
Everything the app can: onboard brands, generate, edit, resize, search the library in plain English, import references, and update brand profiles — via 10 tools on the Streamable HTTP endpoint.
Who owns the generated assets?
You do. Crowd is self-hosted: images are stored on your own disk or S3 bucket, and the code is yours to deploy.
